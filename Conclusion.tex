
\section{Conclusion}\label{sec:Conclusion}
In this report, we presented concrete Haskell implementations of two knowing-how logics: the basic logic $\mathcal{L}_{Kh}$ and its extension with regularity constraints $reg\text{-}\mathcal{L}^U_{Kh}$. We also implemented an explicit model checker with bounded plan length for $\mathcal{L}_{Kh}$, as well as a model checker for $reg\text{-}\mathcal{L}^U_{Kh}$ based on the procedure described in \cite{Demri2023}. Since the original paper does not spell out all technical details of the reachability tests, we made these constructions explicit in our implementation by using depth-first search, which satisfies the time bound. We also incorporated a small optimisation in the implementation of the procedure \texttt{CheckSE}: instead of performing reachability tests inside nested loops, we first compute the set of bad points and then perform the reachability checks. This avoids repeated reachability computations. To support experimentation, we provided parsers for both logics, as well as generators for random models and formulas. These components enabled systematic validation using QuickCheck and HSpec.

We note that the current implementation assumes that all automata provided in the uncertainty component are well-formed. In particular, we do not explicitly verify whether they satisfy the structural conditions required by the theoretical framework. As a result, it is the responsibility of the user to ensure that the automata used in the model conform to these requirements. A natural direction for future work would therefore be to implement a dedicated sanity checker for automata.
